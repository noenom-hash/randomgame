# Toss 미니앱 빌드 가이드 (일반화 버전)

**이 가이드는 모든 Toss 미니앱 프로젝트에 적용 가능합니다.**

---

## 🎯 핵심 해결책: TDSMobileAITProvider 제거

### 문제 증상
```
"서비스를 이용할 수 없어요" 에러 (Toss 앱에서 실행 불가)
```

### 해결 방법
```tsx
// ❌ 잘못된 방식 (브릿지 초기화 실패)
import { TDSMobileAITProvider } from "@toss/tds-mobile-ait";

<TDSMobileAITProvider brandPrimaryColor="#3182F6">
  <App />
</TDSMobileAITProvider>

// ✅ 올바른 방식
<App />
```

**결과**: TDS 브릿지 정상 초기화 ✅

## 📋 프로젝트 초기 설정

### 기술 스택 (필수)
- React 18 + TypeScript
- Vite (빌드 도구)
- `@apps-in-toss/web-framework` (Toss 프레임워크)
- `@toss/tds-mobile` (TDS 컴포넌트)
- `@toss/tds-mobile-ait` (AIT 빌드용)
- React Router DOM (HashRouter)
- CSS Modules 또는 CSS-in-JS

### 빌드 & 배포 명령
```bash
npm run dev      # 로컬 개발 (경고 무시 가능)
npm run build    # .ait 파일 생성
npm run deploy   # 토스 콘솔 배포 (선택사항)
```

## 🔧 필수 설정 파일

### 1️⃣ granite.config.ts

**모든 필수 필드:**
```typescript
import { defineConfig } from "@apps-in-toss/web-framework/config";

export default defineConfig({
  appName: "my-app",                    // 앱 식별자 (영문 소문자, 하이픈)
  brand: {
    displayName: "My App",              // 사용자에게 보여질 이름
    primaryColor: "#3182F6",            // 브랜드 컬러 (16진수)
    icon: "https://static.toss.im/appsintoss/[appId]/[uuid].png",  // Toss CDN URL
  },
  web: {
    host: "localhost",
    port: 5173,                         // 로컬 개발 포트
    commands: {
      dev: "vite --port 5173",          // ⚠️ 반드시 --port 포함
      build: "vite build",
    },
  },
  permissions: [],                       // 필요한 권한 배열 (비어있어도 됨)
  outdir: "dist",
});
```

**⚠️ 주의사항:**
- `appName`: 영문 소문자, 하이픈만 사용 (스페이스 금지)
- `displayName`: 한글/영문 모두 가능
- `icon`: **반드시 Toss CDN URL 사용** (로컬 경로 금지)
  - 형식: `https://static.toss.im/appsintoss/[appId]/[uuid].png`
  - 예: `https://static.toss.im/appsintoss/41891/0bb42336-6d36-4088-97d7-d8c6d579ad1a.png`
  - 로컬 `/app-icon.png` 경로는 Toss 콘솔 제출 시 오류 발생
  - 콘솔에 등록한 앱의 CDN URL을 Toss 지원팀에 요청하여 획득
- `web.commands.dev`는 반드시 `"vite --port <port>"` 형식

### 3️⃣ App.tsx (라우팅 + TDS 브릿지)

**필수 구조:**
```typescript
import { HashRouter, Routes, Route, useLocation } from 'react-router-dom'
import { Asset } from '@toss/tds-mobile'

function AppLayout() {
  const location = useLocation()
  
  return (
    <div>
      {/* 🔴 핵심: TDS 브릿지 초기화 */}
      <div style={{ position: 'absolute', top: -9999, left: -9999, width: 64, height: 64, overflow: 'hidden' }}>
        <Asset.Image
          alt="app logo"
          frameShape={{ width: 64, height: 64 }}
          backgroundColor="transparent"
          src={`${import.meta.env.BASE_URL}app-icon.png`}
        />
      </div>

      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/page2" element={<Page2 />} />
        {/* 라우트 추가 */}
      </Routes>
    </div>
  )
}

export default function App() {
  return (
    <HashRouter>
      <AppLayout />
    </HashRouter>
  )
}
```

**필수 조건:**
- ✅ **HashRouter** (BrowserRouter 금지)
- ✅ **Asset.Image** 렌더링 (TDS 브릿지 초기화용)
- ✅ `frameShape={{ width: 64, height: 64 }}` 정확하게
- ✅ `src`는 public 디렉토리 기준 경로

### 모든 페이지에 `<Top>` 컴포넌트 필수

```typescript

export default function MyPage() {
  return (
    <div>
      <Top
        title={<Top.TitleParagraph size={22}>페이지 제목</Top.TitleParagraph>}
```

## 🎨 CSS/SVG 관련 주의사항

### SVG에서 CSS 변수 금지 ❌

```typescript
// ❌ 작동 안 함
<linearGradient id="gradient">
  <stop offset="0%" stopColor="var(--color-primary)" />
  <stop offset="100%" stopColor="var(--color-dark)" />
</linearGradient>

// ✅ 올바른 방식
<linearGradient id="gradient">
  <stop offset="0%" stopColor="#3182F6" />
  <stop offset="100%" stopColor="#1a64e8" />
</linearGradient>
```

**이유:**
- SVG 요소는 CSS 변수를 인식하지 못함
- 색상값을 직접 하드코딩해야 함

### CSS 변수는 일반 CSS에서만 사용

```css
/* ✅ 가능 */
.button {
  background-color: var(--color-primary);
}

/* ❌ SVG에서는 불가 */
<stop stopColor="var(--color-primary)" />
```

## 🪝 상태 관리: localStorage 훅

### useLocalStorage 구현 (복사용)

```typescript
import { useState } from 'react'

export function useLocalStorage<T>(key: string, initialValue: T): [T, (value: T) => void] {
  const [storedValue, setStoredValue] = useState<T>(() => {
    try {
      const item = window.localStorage.getItem(key)
      return item ? JSON.parse(item) : initialValue
    } catch {
      return initialValue
    }
  })

  const setValue = (value: T) => {
    try {
      setStoredValue(value)
      window.localStorage.setItem(key, JSON.stringify(value))
    } catch (error) {
      console.error(`Error setting localStorage key "${key}":`, error)
    }
  }

  return [storedValue, setValue]
}
```

### 사용 방법 (타입 명시 필수)

```typescript
// ✅ 올바른 사용
const [count, setCount] = useLocalStorage<number>('count', 0)
const [items, setItems] = useLocalStorage<Item[]>('items', [])
const [userData, setUserData] = useLocalStorage<User>('user', defaultUser)

// ❌ 타입 없음 (피하기)
const [count, setCount] = useLocalStorage('count', 0)
```

## ⚠️ 주의해야 할 점

### 1. 로컬 개발 vs 토스 앱 배포

```
npm run dev 실행 시:
┌─────────────────────────────────────────────┐
│ Failed to find Response internal state key   │
│ ⚠️ 이 경고는 무시해도 됨 (Granite 내부)     │
│ 로컬에서 정상 작동하면 토스도 정상          │
└─────────────────────────────────────────────┘
```

### 2. 타입 체크

```bash
# 배포 전 반드시 실행
npx tsc --noEmit
```

### 3. 번들 크기 모니터링

```
목표: < 500KB (gzip)
경고: 500KB 이상 시 청크 분할 필요
```

### 4. 라우터 변경 금지

```typescript
// ❌ 절대 금지
import { BrowserRouter } from 'react-router-dom'
<BrowserRouter>

// ✅ 필수
import { HashRouter } from 'react-router-dom'
<HashRouter>
```

**이유:**
- Toss WebView의 초기 URL이 `/`가 아닐 수 있음
- HashRouter는 `#/`로 동작하므로 항상 안전

### 5. public 폴더 파일

```
public/
├── app-icon.png          # ⭐ 필수 (granite.config.ts의 icon)
└── 기타 정적 파일
```
