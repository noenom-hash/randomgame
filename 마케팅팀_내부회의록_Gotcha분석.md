## 6️⃣ SWOT 분석: Gotcha vs RandomGame

**리더**: "이제 정리해보자. Gotcha와 우리의 경쟁 구도를."

### 🔵 Gotcha 강점

```
1. 대단한 다운로드 수 (5백만)
2. 신뢰할 수 있는 평점 (4.53)
3. 풍부한 기능 세트 (다양성)
4. 확립된 사용자 베이스 (초기 성공)
5. 여러 플랫폼 지원 (iOS/Android)
```

### 🔴 Gotcha 약점

```
1. 인터넷 필수 (광고 때문)
   → 오프라인 사용 불가
   → 모바일 신뢰도 떨어짐

2. 높은 복잡도
   → 신규 사용자 진입장벽
   → "어떻게 하는 거죠?" 리뷰 있음

3. 목적이 불명확
   → "뭐를 하는 앱인가?"
   → 범용 도구라서 콘셉트 약함

4. 월간 활성 사용자 감소?
   → 5백만 총 / 월 44,000
   → 연간 기준 아마 70만 정도?
   → DAU 추정: 20만 이하?

5. 30일 이탈율 95%
   → Casual 게임 평균
   → 즉, "한 번 재미본 후 버려지는" 앱
```

**팀원 A**: "아, Gotcha도 생각보다 약점이 많네."

**리더**: "맞아. 이제 우리의 기회를 봐보자."

### 🟢 RandomGame 기회

```
1. 오프라인 1순위
   → Gotcha는 인터넷 필수
   → 우리는 100% 오프라인
   → 배터리 사용량도 적음

2. 명확한 목적
   → "밥 결정 게임"
   → 사용 시나리오가 구체적
   → "언제 써야 할까?"가 명확

3. 극도로 단순한 UX
   → 3초 이해
   → 신규 사용자 온보딩 쉬움
   → "어떻게 하는 거죠?" 리뷰 없을 것

4. 한국 시장 타게팅
   → Gotcha: 글로벌 (한국 버전도 있지만 번역 수준)
   → 우리: 한국 시장 특화
   → 밥/커피 문화와 매칭

5. 자연 바이럴
   → 결과 재미있어서 촬영/공유 유도
   → Gotcha는 결과가 "누가 선택됐어?"만 함
```

### 🔴 RandomGame 위협

```
1. Gotcha의 기존 5백만 사용자
   → 경쟁 우위

2. 복잡한 기능 vs 단순한 기능
   → 사용자는 "더 많은 기능"을 좋아함
   → "반복성"이 부족할 수 있음

3. 30일 이탈율
   → 우리도 피할 수 없는 문제
   → Phase 2/3에서 해결해야 함
```

   ✓ Phase 3: 글로벌화 (온라인)

5. 리스크
   ⚠️ 30일 이탈율 95% (캐주얼 게임 숙명)
   ⚠️ Gotcha 사용자 학습 효과
